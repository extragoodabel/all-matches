TECH IMPLEMENTATION REQUIREMENTS (Matter.js)
Use Matter.js. Keep it light and efficient.

1) Render approach
- Use Matter.Engine and Matter.Render to draw to a single full-screen canvas.
- Position the canvas as fixed, top:0 left:0, width:100vw height:100vh, pointer-events:none, high z-index.
- The DOM elements for logo card, tagline card, and chyron should be real HTML layered above or below the canvas depending on the phase, but the logo must visually appear “in the ocean”.
  Preferred: keep logo as HTML but sync its transform to a Matter body so it truly floats with physics.

2) Emoji bodies
- Each emoji is a circle body (Matter.Bodies.circle).
- Use restitution low to medium (slight bounce), friction moderate, frictionAir moderate for damping.
- Gravity ON.
- Create enough bodies to fill the screen. Aim 140 to 240 emoji bodies depending on device width.
- Use 2 to 3 depth layers by varying emoji size and alpha. Keep it simple. Do NOT blur heavily.

3) Cohesion / soft clustering
- Add loose constraints between nearby emoji bodies to mimic “water molecule cohesion”.
- Use Matter.Constraint with low stiffness and some damping.
- You do not need constraints for every pair. Do a sparse set:
  - For each emoji, connect it to 1 to 3 nearest neighbors at spawn time.
- This produces the bonded-but-fluid mass.

4) Flood-in spawn behavior
- Spawn most emoji bodies offscreen left and offscreen right in two clusters.
- Give them an initial inward velocity so they surge toward center.
- Let physics handle collision and packing.

5) Drain behavior
- Implement drain by opening the bottom boundary:
  - Either remove the floor body entirely
  - Or create a gap in the floor center and apply a downward attractor force in that region
- When drain starts, ensure emojis exit bottom within the allotted time (1 second). You can increase gravity slightly during drain for speed.

6) Logo physics integration
- Create a Matter rectangle body that represents the logo card.
- Sync the HTML logo element position and rotation to this body every tick.
- Give it density lower than emojis so it floats and bobs.
- When drain starts, allow it to get pulled down with the emojis and exit.

7) Performance
- Must run smoothly on mobile.
- Cap engine updates at requestAnimationFrame.
- Use devicePixelRatio scaling but clamp DPR to 2 for performance.
- Clean up the engine, render, runner, and event listeners on unmount.
- Use a single animation sequence. It can be reused across app loads, but it is fine to generate each time if simpler.
